Problem Statement

Inefficient Dorm Checking & Live Status Monitoring
Alert Center with clear patient location

03
Lengthy patient medical records, spending too much time in both recording and reviewing
Digitalize and automate the medical records with diagrams that help quicken the review process

Goals & Metrics

UX Design

Set function focus and simplify work flow

Information source is diverse in this project. PM provides information from UNEO, while the salesperson represent the medical stakeholders. I need to be objective to create an efficient design with empathy.
UX Design

Responsive Design based on tasks

As mentioned above, there are distinct responsibility for devices with different resolution size. We will need clear preferences on UI design for different sections in terms of using scenarios.
UX Design

Create browser-compatible web-app (front-end)

As they are creating a web-app, I also need to make sure the front-end of it is compatible across popular browsers and ready for back-end developers to create linkage with their database.

UX Planning

Stage 1: User Mission Analysis, Userflow Design, Sitemap

Identity tasks and key userflows

The first step is to identity all the tasks to be performed in each page, under different possible and foreseeable situations.
We can also clear some covered misunderstandings towards functionality and workflow expectations.

Unwanted functions - Alerts & Alarms

During the UX planning stage, through back-and-forth discussion, we identified several adjustments over userflows and unwanted functions as well. It helps us to save lots of revision time.
I thought nurses would need to save logs and tasks to follow up within the app.
However, the sales from UNEO gave me feedbacks from medical side that nurses would use their own systems to do that, so we deleted the function and revise the userflow.

Design Style

Stage 2 - Stylescapes, Style Previews
I provide UNEO several style explorations. I usually provide both stylecapes or moodbaords and style preview for clients. Stylecapes (preferred) help explain how the style fits in the brand level and gives a bigger picture so that they could understand the style and tone better.
Together with the style preview, I created a few common UI elements to demonstrate how the style and tone would be applied in brief. It helps UNEO to understand the style in a very applicable way too.
